# Vision

**Open-source, versioned {{DOMAIN}} legislation — structured for humans, Git, search, and AI.**

This project does not try to become a replacement for official legal sources. Official sources remain authoritative.

We are building the infrastructure that official portals usually do not provide in a Git-native way:

- Markdown files that humans can read and review.
- Git history for every structural change.
- Diffs between versions of an act.
- Machine-readable metadata for search, validation, and automation.
- Stable source links and import provenance.
- A neutral contribution workflow for developers, practitioners, researchers, and AI builders.

## Why this exists

{{DOMAIN}} regulation is spread across laws, government decisions, ministerial orders, technical norms, authority procedures, and guidance. People who work in this domain need reliable structure around those sources.

This repository aims to provide that structure without pretending to replace the official text.

## What can be built on top

If the source material is structured consistently, others can build:

- AI legal assistants with source citations.
- Git diffs between legislative versions.
- Notifications when a relevant act changes.
- Semantic search over domain rules.
- RAG systems for practitioners, consultants, and researchers.
- Automated compliance checklists.
- Stable article-level citations.
- Open APIs and datasets for research.
- Teaching material for universities and professional communities.

## Principles

- Official sources first.
- Metadata before interpretation.
- Provenance for every import.
- No legal advice.
- No fake or hallucinated articles.
- No third-party copyrighted annotations unless reuse rights are explicit.
- Small, reviewable contributions.
- Open governance and neutral positioning.

## OCKI Philosophy

This repository follows the Open Construction Knowledge Infrastructure (OCKI) principles:

- **Open**: MIT license for structure, tooling, and original content.
- **Versioned**: every change tracked in Git with clear commit messages.
- **AI-readable**: consistent Markdown and JSON structure for retrieval systems.
- **Provenance-first**: every official text import records source, date, and method.

OCKI-compliant repositories are designed to be federated: they share the same schema, the same validation scripts, and the same contribution conventions, so knowledge can be linked across domain boundaries.
