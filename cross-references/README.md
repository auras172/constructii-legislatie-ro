# cross-references/

Auto-detected cross-reference suggestions generated by `scripts/detect-cross-references.mjs`.

> **These are suggestions only.** No metadata files are modified. Human review is required before acting on any finding.

## Files

### `cross-references-raw.json`

Every individual reference match found in the official text blocks, with:

- `source_slug` ‚Äî which act file contains the reference
- `source_line` ‚Äî line number within the OFFICIAL_TEXT_START/END block
- `matched_text` ‚Äî the exact text that matched
- `act_type` ‚Äî normalized type (lege / hotarare / ordonanta / ordin / normativ)
- `number` and `year` ‚Äî extracted from the match
- `article_ref` ‚Äî if an article reference (e.g. `art-7`) follows immediately on the same line
- `suggested_target_slug` ‚Äî the repository slug this match resolves to, or `null`
- `status` ‚Äî `"resolved"` if the target slug exists in `metadata/acts/`, else `"unknown"`

### `relationships-auto.json`

Aggregated view per source act:

- `references_in_text` ‚Äî sorted list of resolved target slugs found in the text
- `unresolved_references` ‚Äî matched text strings that could not be mapped to a known slug
- `article_level_refs` ‚Äî resolved references that also had an adjacent article reference

### `relationships-diff.md`

Human-readable comparison between auto-detected references and the existing `related_acts` arrays in `metadata/acts/*.json`. Sections per act:

- **Already in related_acts** ‚Äî detected and already recorded (no action needed)
- **Detected in text but NOT in related_acts** ‚Äî suggested additions with line numbers
- **In related_acts but NOT detected in text** ‚Äî relationships in metadata not found by text scan (may be intentional ‚Äî e.g. general context relationships)
- **Unresolved** ‚Äî references detected but not matching any known act slug

## How to use

1. Read `relationships-diff.md` ‚Äî it is the primary human review document.
2. For each "suggested addition", verify the reference in the source text is meaningful (not just a historical citation in a preamble).
3. If confirmed, manually add the slug to `related_acts` in the relevant `metadata/acts/<slug>.json` file.
4. Re-run `node scripts/validate-metadata.mjs` after any metadata edits.

## Regenerating

```sh
node scripts/detect-cross-references.mjs
```

Run from the repository root. Overwrites all three output files. Does not modify any other files.

## Intentional non-actions

- No metadata files are modified.
- No legal text files are modified.
- The script is not run in CI (it is a generation/suggestion tool, not a validator).
- Metadata-only acts (no official text imported) are skipped.
- Self-references (an act referencing itself) are skipped.
